# Migration Notes — Canvas Free for Teachers → GitHub Pages

> Instructure permanently discontinued **Canvas Free for Teachers (FFT)**. This document records every Canvas feature we used, what it was replaced with, why, and any compromises.

---

## Feature-by-feature mapping

| Canvas feature | GH-Pages replacement | Why this choice | Compromises |
|---|---|---|---|
| **Course shell** | MkDocs Material site at `https://densesparse.github.io/python-pathways/` | Free; markdown-native; gorgeous default theme; native search; the gold standard for course/docs sites | The site is **public**. Course content is curriculum, so this is fine and helps recruiting. PII is kept off the public site. |
| **Course sections (Track A/B/C)** | Track-prefixed nav tabs in `mkdocs.yml` + per-track index pages | Same effect (track-specific paths), simpler | No automatic enrollment-based hiding; students follow only their track by convention |
| **Module unlock by date** | Client-side JS (`course/assets/js/unlock.js`) using a per-page `<script id="unlock-config">` | Honor-system gating that's lightweight + easy to override for instructor preview | Bypass-able by disabling JS or `?preview=1`; acceptable for a 15-student trust-based cohort |
| **Auto-graded quizzes (timed, no backtrack)** | Interactive HTML quiz pages built from YAML by `scripts/build_quizzes.py`; uses `course/assets/js/quiz.js` for timer, shuffle, no-backtrack, instant scoring | Genuinely good UX (matches Canvas's New Quizzes feel); fully self-contained, no server | Score is computed client-side and the student self-reports via a Google Form; honor system on whether they actually closed their notes |
| **Assignment submission with due date + comments** | One Google Form (free) → backed by a Google Sheet | Free; reliable; students get an email confirmation; instructor can grade in the Sheet | No deadline auto-cutoff (Forms can be "closed" manually each week if needed) |
| **Discussion forum per section** | **giscus** (GitHub Discussions) for Tracks B & C; **Padlet** (free) or curated email-driven page for Track A | Track C is already on GitHub. Track B has Google. Track A 5th-graders can't have GitHub accounts (COPPA), so Padlet's password-locked board fits | Setup is per-track |
| **Gradebook with weighted columns** | Private Google Sheet (`Roster / Quizzes / Assignments / Capstone / Wallet` tabs) | Free; familiar; safer for PII | No live-link to the public site; instructor edits `grades.yml` in the repo and runs `update_wallet.py` to publish per-student summaries |
| **Per-student "Progress Wallet" page** | Per-student markdown at `course/wallets/{uuid_slug}.md`, regenerated weekly | Each wallet has an unguessable URL; only the student knows theirs | Public site (so URL discovery would expose someone else's wallet). 12-char UUID slugs make this brute-force-infeasible for the cohort size |
| **Certificate of completion (file upload)** | Per-student HTML at `course/certificates/{slug}.html`, generated by `scripts/make_cert.py` from the existing 3 templates | Templates already designed; easy print-to-PDF | Same UUID-slug privacy posture |
| **Welcome announcements** | Markdown files under `course/news/` + a homepage banner | Stays in version control, clearer than Canvas's announcements UI | No push notifications; instructor emails the announcement separately |
| **Email invitations** | Done by hand in Gmail using the templates in [Artifact 3](03_invitation_emails.md) | Same as Canvas under the hood — Canvas was just sending email | None |
| **Files (asset hosting)** | Static files served by GH Pages from `course/assets/` and per-week module directories | Free; CDN-backed | None |
| **Observer (parent) role** | We just send wallet/cert URLs to parents directly + the parent onboarding doc | Simpler | No "live observer view" of student progress; weekly wallet update is the equivalent |
| **Student-private content** | UUID-slug URLs + private Google Sheet for grades | Defense-in-depth: per-student summary on the public site (slugged), real grades only in the private Sheet | Public site is by-design fully visible; that's a feature not a bug |

## What got better in the migration

- **Source-controlled curriculum.** Every change is a git commit. You can re-run the entire course in 2027 by branching `main`, updating dates, and pushing.
- **Open content = recruiting.** Future cohorts find the syllabus by Googling. Canvas hid everything behind a login.
- **Real engineering tools for Track C.** Pros are submitting GitHub repos, taking quizzes that timer-out, and joining GitHub Discussions — all skills they'll use professionally.
- **Better search.** MkDocs Material's built-in search beats Canvas's search by a wide margin.
- **Beautiful theme.** The course actually *looks* like a course.
- **Cost.** Canvas FFT was free; this is also free; future-proof from another vendor sunset.

## What got worse in the migration

- **No native auto-emails on assignment due dates.** Mitigation: instructor sends a manual reminder Sunday night via Gmail (1 minute of work).
- **No "drop-box" for binary file uploads.** Mitigation: Replit/Colab/GitHub links are pasted into the Google Form. For the rare case a student needs to upload an image, they put it in a Google Drive folder and share the link.
- **No native parent-observer role.** Mitigation: parents get the wallet URL + weekly update emails. Honestly, it's clearer.
- **Honor-system quizzes.** Mitigation: per-quiz dollars are small ($1); the capstone is the real assessment. Cheating to inflate quiz score is poor strategy.

## If something goes wrong

**Issue:** site doesn't deploy.
- Check the **Actions** tab on GitHub for the failed workflow run.
- Most common failure: a broken markdown link → `mkdocs build --strict` flags it. Fix locally, push.

**Issue:** student says their wallet shows wrong totals.
- Check `grades.yml` for that student's row.
- Re-run `python scripts/update_wallet.py --student "First Last"`, commit, push.

**Issue:** student lost their wallet URL.
- Run: `grep "First Last" roster.csv` → get their slug → email them `https://densesparse.github.io/python-pathways/wallets/{slug}/`.

**Issue:** quiz JavaScript doesn't run.
- Check that `course/assets/js/quiz.js` is being served (open dev tools → network tab).
- The quiz HTML pages are built by `scripts/build_quizzes.py` — re-run if the file is missing.

**Issue:** Need to revoke a student's access.
- Email the student that their wallet URL is invalid; rotate the slug in `roster.csv` (regenerate via `update_wallet.py --new`); re-run `update_wallet.py`. The old URL now 404s.

**Issue:** Discussion forum spam.
- giscus uses GitHub Discussions; spammers are extremely rare and easy to ban. Padlet has built-in moderation (turn it ON).
